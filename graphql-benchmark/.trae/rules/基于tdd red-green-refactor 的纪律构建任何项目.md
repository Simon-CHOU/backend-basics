TDD（Test-Driven Development，测试驱动开发）

1. 编写测试用例
2. 编写代码使测试通过
3. 优化代码
4. 重复以上步骤，直到项目完成
5. McCabe（Cyclomatic Complexity）超过20时，需要进行代码重构
6. 使用jacoco进行代码覆盖率分析
8. 行覆盖率 ≥ 85%
9. 分支覆盖率 ≥ 80%
10. 方法覆盖率 ≥ 90%
11. 禁止刻意编写注水测试、为凑覆盖篡改业务逻辑
12. 兜底异常、日志分支、灰度配置分支 允许不计入强制覆盖，做人工评审豁免
13. 提交代码前，需要进行代码格式化lint
14. 遵守 Google Code Style Guide
15. 单元测试全部通过前，不要进行 e2e 测试
16. 使用playwright进行 e2e 测试
